# Release Notes - v2025.12.1

## Overview

This patch release addresses critical issues identified in v2025.12, focusing on essential bug fixes, security improvements, and stability enhancements. This update ensures better reliability for development workflows and production deployments.

## üîß Bug Fixes and Improvements

### Setup Script Enhancements

#### PowerShell Version Validation
- **Issue**: Setup script could run with incompatible PowerShell versions, leading to unexpected failures
- **Fix**: Added PowerShell version checking to ensure compatibility before script execution
- **Impact**: Prevents setup failures on systems with older PowerShell versions

### Security and Performance Enhancements

#### Rate Limiting Implementation
- **Enhancement**: Implemented rate limiting for manifest endpoints to prevent abuse
- **Security**: Protects manifest API endpoints from excessive requests
- **Performance**: Improves overall system stability under load
- **Implementation**: Added configurable rate limiting with proper error handling

### Development Experience Improvements

#### Enhanced Error Logging
- **Issue**: Inconsistent error and success message formatting in ItemCRUDController
- **Fix**: Standardized logging format for better debugging experience
- **Impact**: Developers now get clearer, more actionable error messages

#### DataHub Wizard Integration Fix
- **Issue**: Missing `hostDetails` parameter in `callDatahubWizardOpen` function calls
- **Fix**: Added proper `hostDetails` configuration to DataHub wizard calls
- **Impact**: DataHub integration now works correctly with proper host context

### OneLake Component Fixes

#### Path Resolution Bug Fix
- **Issue**: OneLakeViewController was using incorrect path resolution logic
- **Root Cause**: Path handling inconsistencies between relative and absolute paths
- **Fix**: Corrected path resolution in OneLakeViewController
- **Impact**: OneLake file navigation and operations now work reliably
